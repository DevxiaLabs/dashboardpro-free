# Contributing to DashBoard Pro Free

Thanks for your interest in contributing! Please follow these guidelines to help us keep things running smoothly.

- Fork the repository and create your branch from develop: git checkout -b feat/your-feature
- Make your changes with clear, focused commits
- Follow code style: Prettier + ESLint, TypeScript strict, Tailwind utility-first
- Run linters and build locally before pushing
- Write/update tests when applicable
- Open a pull request to develop with a clear description and checklist
- Link related issues (e.g., closes #123)

## Project setup
- Framework: Next.js (App Router), React, TypeScript
- Styling: Tailwind CSS
- Package manager: pnpm

## Commit messages
Use conventional commits when possible: feat:, fix:, docs:, chore:, refactor:, perf:, test:

## Pull request checklist
- [ ] Lint passes (pnpm lint)
- [ ] Build succeeds (pnpm build)
- [ ] Screenshots for UI changes
- [ ] Updated docs if needed

## Code of Conduct
Be respectful and inclusive. We follow the GitHub Community Guidelines.
